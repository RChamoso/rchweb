Mis primeros pasos en HTML5 - Trasteando con el Canvas

RSS Link
Hernán Alijo Blog

Send Private Message Posted December 15, 2011 9:27 am

 

En este artículo firmado por Paul Farquhar (@TheGlenpaul), vamos a poder conocer un poco más cómo poder crear y divertirnos con Canvas en HTML5. Veréis que ilustrativo y qué detallado lo muestra!!  

 

Introducción

Yo he escrito algo de código en mi vida. Los lenguajes de programación que he utilizado son: Cobol (bufff), Pascal, C, C + + Java, y otros no tan conocidos como Algol y Smalltalk 80. Nunca he hecho mucho con HTML, pero HTML5 está siendo empujado a través de los medios.

¿Tal vez llegado el momento de echar un vistazo más de cerca?

 

¿Cómo fue la primera impresión con HTML?

La primera vez que vi HTML / Javascript no me gustó (a partir de aquí, al decir HTML me referiré al conglomerado de HTML, JavaScript / ECMAScript y CSS). Yo no podía depurar las cosas y que tenía que hacer prueba y error o guardar mensajes e en el proceso. Todo parecía un gran lío para mí.

Mi último vistazo a HTML5 fue mucho más agradable!

Con el navegador Chrome tengo un depurador - lo que realmente depura el código completo. También puedo establecer puntos de interrupción en los acontecimientos. La "herramienta de desarrollo" también me da un inspector para los elementos y los estilos utilizados. Puedo ver los distintos recursos (o activos) que se utilizan y más elementos que no he usado en profundidad, como la supervisión del rendimiento. El desarrollo en HTML ha recorrido un largo camino desde que lo probé. En la lectura de algunos artículos sobre HTML5, el Canvas sale con buenas críticas. El Canvas permite a los desarrolladores web dibujar dinámicamente gráficos en la pantalla sin ningún tipo de plug-ins, lo que antes no era posible hasta HTML5. Me decidí a escribir algo para tratar con el Canvas.

 

Pero primero necesitaba una idea…

Mirando alrededor de la oficina de mi casa, mi caja de Spirograph sobresalía de un estante. Mi inspiración para escribir una aplicación sobre un espirógrafo!


Como puedes ver, la mayoría de los bits todavía están allí y me sorprendió encontrar una pila de viejos dibujos que hice cuando era niño:


Si no sabes qué es un espirógrafo, es básicamente un anillo plano de plástico con algunas ruedas dentadas. El anillo se fija en una hoja de papel encima de un cartón. Después, se coloca una rueda dentada en el interior del anilo, colocando dentro de él la punta de un bolígrafo. A continuación, se echaba a rodar con el boli, dando lugar a interesantes y originales figuras. Yo solía hacer durante horas y horas. Aquí hay un enlace para conocer un poco más su historia.

El espirógrafo es la matemática aplicada sobre la base de cicloides. Si el engranaje se mueve alrededor de la parte exterior del anillo se obtiene una Epitrocoide, trasladado al interior del anillo se obtiene una Hipotrocoide.

En mi aplicación de prueba pretendo dibujar las curvas de tipo Hipotrocoide.

 

Diseño inicial

Quiero una interfaz simple que permita al usuario:

  • Entrar en el radio del anillo y la rueda
  • Introducir la distancia entre el lápiz y el centro de las ruedas
  • Algunos botones para cambiar fácilmente los números de arriba (+ -10, + -1)
  • Un botón para empezar a dibujar y un botón para borrar el dibujo

También voy a añadir algo de código para prepararse para el cambio del color de la pluma y el grosor del lápiz. Esto puede ser una futura ampliación.

Este es el sitio de donde me basé para los principios matemáticos: Mathematische Basteleien. El autor tiene una buena explicación de las matemáticas involucradas.

El siguiente paso es escribir todo dentro Para la edición utilicé Notepad + + y esta es la primera creación de Sprio:

 

spiro1.html

 

<!DOCTYPE html>
<html>
 <head>
 <title>Spiro-01</title>
  <script type="application/javascript">
    function DrawSpiro() {
        var objCanvas = document.getElementById("canvas");
        var ctx = objCanvas.getContext("2d");
        ctx.save();

        var size = 0;
        if(objCanvas.width<objCanvas.height)
            size = objCanvas.width;
        else
            size = objCanvas.height;
        ctx.translate(size/2,size/2);
        
        var OuterRadius = document.SpiroInput.OuterRadius.value;
        var InnerRadius = document.SpiroInput.InnerRadius.value;
        var PenOffset = document.SpiroInput.PenOffset.value*InnerRadius/100;
        var PenColour;        // for later extension
        var PenThickness;    // for later extension
        var StartX;            // we start drawing here
        var StartY;            // we start drawing here
        var x,y;
        var Step = Math.PI/180;
        var Angle = 0;
        
        StartX = (OuterRadius-InnerRadius)*Math.cos((InnerRadius / OuterRadius)*Angle)+PenOffset*Math.cos((1 - (InnerRadius / OuterRadius))*Angle);
        StartY = (OuterRadius-InnerRadius)*Math.sin((InnerRadius / OuterRadius)*Angle)-PenOffset*Math.sin((1 - (InnerRadius / OuterRadius))*Angle);

        ctx.beginPath();
        ctx.moveTo(StartX,StartY);
        do
        {
            Angle += Step;
            x = (OuterRadius-InnerRadius)*Math.cos((InnerRadius / OuterRadius)*Angle)+PenOffset*Math.cos((1 - (InnerRadius / OuterRadius))*Angle);
            y = (OuterRadius-InnerRadius)*Math.sin((InnerRadius / OuterRadius)*Angle)-PenOffset*Math.sin((1 - (InnerRadius / OuterRadius))*Angle);
            ctx.lineTo(x,y);
            ctx.stroke();
        } while ((x != StartX) && (y != StartY));

        ctx.restore();
    }
    
    function clearCanvas(){
        var canvas = document.getElementById("canvas");
        var context = canvas.getContext("2d");
        context.clearRect(0, 0, canvas.width, canvas.height);
    }

    function OuterP10(){
        document.SpiroInput.OuterRadius.value = String(Number(document.SpiroInput.OuterRadius.value)+10);
    }
    function OuterP1(){
        document.SpiroInput.OuterRadius.value = String(Number(document.SpiroInput.OuterRadius.value)+1);
    }    
    function OuterM10(){
        document.SpiroInput.OuterRadius.value = String(Number(document.SpiroInput.OuterRadius.value)-10);
    }
    function OuterM1(){
        document.SpiroInput.OuterRadius.value = String(Number(document.SpiroInput.OuterRadius.value)-1);
    }
    function InnerP10(){
        document.SpiroInput.InnerRadius.value = String(Number(document.SpiroInput.InnerRadius.value)+10);
    }
    function InnerP1(){
        document.SpiroInput.InnerRadius.value = String(Number(document.SpiroInput.InnerRadius.value)+1);
    }    
    function InnerM10(){
        document.SpiroInput.InnerRadius.value = String(Number(document.SpiroInput.InnerRadius.value)-10);
    }
    function InnerM1(){
        document.SpiroInput.InnerRadius.value = String(Number(document.SpiroInput.InnerRadius.value)-1);
    }
    function PenOffsetP10(){
        document.SpiroInput.PenOffset.value = String(Number(document.SpiroInput.PenOffset.value)+10);
    }
    function PenOffsetP1(){
        document.SpiroInput.PenOffset.value = String(Number(document.SpiroInput.PenOffset.value)+1);
    }    
    function PenOffsetM10(){
        document.SpiroInput.PenOffset.value = String(Number(document.SpiroInput.PenOffset.value)-10);
    }
    function PenOffsetM1(){
        document.SpiroInput.PenOffset.value = String(Number(document.SpiroInput.PenOffset.value)-1);    
    }
  </script>
 </head>
 <body>

    <button onclick="InnerM10();">-10</button>
    <button onclick="InnerM1();">-1</button>
    <button onclick="PenOffsetM10();">-10</button>
    <button onclick="PenOffsetM1();">-1</button>

Vamos a echar un vistazo a algunos bits del código

El Canvas se define a sí mismo en la línea 90:

 

canvas id="canvas" width="400" height="400"></canvas>

 

Todos los dibujos que suceden en el DrawSprio function () comienzan a partir de la línea 6.

El primer paso para trabajar con el Canvas es tener un contexto para trabajar. Todas las funciones de dibujo necesitan de un contexto (no trabajan sobre el Canvas en sí). Mi contexto se llama ctx:

 

var objCanvas = document.getElementById("canvas");
var ctx = objCanvas.getContext("2d");
ctx.save();

 

Lo que estoy haciendo aquí, además, es ahorrar el contexto actual. Esto me permite trabajar con seguridad y restaurar el viejo contexto con el que mi función termina (con ctx.restore ()). No es necesario para esta aplicación simple, pero creo que es algo bueno acostumbrarse para futuros desarrollos más complejos.

Las matemáticas requieren del punto de origen para estar en el centro del área de dibujo. Así que tengo que traducir esto al Canvas. En primer lugar busco el lado más pequeño y luego, mover el centro hacia el centro del lienzo:

 

if(objCanvas.width<objCanvas.height)
    size = objCanvas.width;
else
    size = objCanvas.height;
ctx.translate(size/2,size/2);

 

Dibujar líneas en Canvas consiste se denomina como ‘path’. Un ‘path’ es esencialmente un polígono formado por líneas y arcos. También puede tener varios caminos secundarios. Un contexto sólo tiene un camino al mismo tiempo. Para extraer algunas líneas que necesitas para construir un trazado con funciones tales como lineTo y moveTo, puedes cambiar los elementos como el color y el grosor de la línea y, finalmente, utilizar tiempos para dibujar el trazado. Los próximos pasos en el código son muy sencillos:

  • calcular el punto de partida (el = startx y líneas StartY =) matemáticamente.
  • comenzar un ‘path’ con ctx.beginPath ();
  • mover la pluma a ese punto: ctx.moveTo (startx, StartY);

El siguiente es el bucle principal que termina cuando se vuelve al punto de partida. Todo lo que hace es calcular el siguiente paso (con las mismas matemáticas que para el punto de partida) e ir a ese punto: ctx.lineTo (x, y); Esto no trae ninguna línea en la pantalla sino que añade la línea a la ruta. Para que sea visible lo que necesito, es esta función: ctx.stroke (); La función Canvas relacionada es clearCanvas (), que hace precisamente esto:

  • obtener un contexto
  • Rectángulo: context.clearRect (0, 0, canvas.width, canvas.height);

Las demás funciones que manejan los botones -10 + y + -1 son para cambiar los parámetros.

La aplicación funcionó, pero no estaba satisfecho.

 

Esto necesita un poco de optimización

Sí que funciona, pero:

  • es muy lento
  • el botón de funciones + -10 y + -1  no son óptimos
  • el circuito principal tiene una gran cantidad de matemáticas que se pueden optimizar

Antes de empezar a optimizar la aplicación por primera vez se midió el tiempo que se tarda en dibujar una curva utilizando la herramienta de perfiles de Chrome, tomando el tiempo que necesita DrawSpiro:

  •  Parámetros de la prueba: 175, 50, 130
  • Sprio1: 810ms para DrawSpiro

Primer paso: mover la llamada a la función carrera fuera del circuito principal:

 

} while ((x != StartX) && (y != StartY));
ctx.stroke();
ctx.restore();

 

  • Sprio1-1: 7ms

Una gran diferencia! Un camino grande es mucho mejor que muchos pequeños.

 

Segundo paso: optimizar algunos de los cálculos en el bucle principal. Algunas de las afirmaciones nunca cambian y se podrían calcular una vez fuera del bucle:

 

var OmI = OuterRadius-InnerRadius;
var IdO = InnerRadius / OuterRadius;
var IdO1 = 1 - IdO;
…
StartX = OmI*Math.cos(IdO*Angle)+PenOffset*Math.cos(IdO1*Angle);
StartY = OmI*Math.sin(IdO*Angle)-PenOffset*Math.sin(IdO1*Angle);

 

Necesitaba un poco de los parámetros de la nueva prueba para obtener un tiempo de dibujo más largo:

  • Parámetros de la prueba: 181, 47, 130
  • Spiro1-1: 310ms la primera vez y 190ms en las siguientes ejecuciones.
  • Spiro1-2: 180ms la primera vez, y cada 43ms se ejecutan otros.

Una vez más, una actuación de gran impulso. Esto demuestra que JavaScript no puede reconocer las porciones de código que no cambian en el circuito y hacer de ellas una constante. Esta es una de las principales diferencias entre un lenguaje dinámico interpretado y un lenguaje compilado que en el optimizador puede hacer maravillas. Curioso es que las segundas ejecuciones y siguientes son más rápidas que la primera. No tengo una explicación para ello - ¿tal vez vosotros sí? Otra área para acelerar la aplicación de la resolución es el cálculo de los pasos. En lugar de cada grado (Paso = Math.PI/180) cada 10 grados es suficiente (Paso = Math.PI/180 * 10).

  • Spiro1-2: 128 ms en frescos de carga, segunda prueba de 5 ms

Ahora tengo la velocidad que buscaba. El siguiente paso es limpiar el botón + -10 y + -1. Empecé con una función más por menos (sólo dos botones y una función que se muestra aquí):

 

<button onclick="OuterP10();">+10</button>
<button onclick="InnerP10();">+10</button>

function OuterP10()
{
   document.SpiroInput.OuterRadius.value = 
      string(Number(document.SpiroInput.OuterRadius.value)+10);
}

 

Todo esto se puede escribir más elegante:


<button onclick="ChangeParam('O',10);">+10</button>
<button onclick="ChangeParam('I',10);">+10</button>

function ChangeParam(param,value)
{
    switch(param)
    {
    case "O": document.SpiroInput.OuterRadius.value  =  
        String(Number(document.SpiroInput.OuterRadius.value)+value);break;
    case "I": document.SpiroInput.InnerRadius.value  = 
        String(Number(document.SpiroInput.InnerRadius.value)+value);break;
    case "PO": document.SpiroInput.PenOffset.value = 
        String(Number(document.SpiroInput.PenOffset.value)+value);break;
    }
}

 

Ahora tengo una pequeña aplicación muy satisfactoria. El formato de los botones y el texto circundante podría ser más claro, y estaría bien cambiar el color del lápiz y el grosor – que será una extensión, o lo puedes añadir tú por tu cuenta.
